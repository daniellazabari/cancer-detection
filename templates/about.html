About the project

Leveraging the power of data science and machine learning, this project uses an ensemble of machine learning models to predict breast cancer diagnoses. It's designed with user-friendly web interfaces, deployed within a Docker container for easy scalability and reproducibility.

About the Dataset

The web-app uses the Breast Cancer Wisconsin (Diagnostic) DataSet, obtained from Kaggle, which contains features computed from a digitized image of a fine needle aspirate (FNA) of a breast mass and describe characteristics of the cell nuclei present in the image.

The Model

The backbone of the Breast Cancer Prediction Project is an ensemble of various machine learning algorithms - Logistic Regression, Decision Trees, and Support Vector Machines - bound together using a Voting Classifier.
I chose an ensemble approach to leverage the diversity of these models and to improve the generalization and robustness of predictions. The models were trained and evaluated using an 80-20 train-test split strategy and achieved an accuracy of 97.3% on the test set.
Additionally, the project employs a pipeline that bundles the preprocessing and modeling steps together so that the whole bundle can be done in a single step. I choose to use a pipeline to ensure cleaner and more concise code, and to make the project easier to productionize.

The Web App and Docker Deployment

The web app is built using Flask and deployed within a Docker container. The app is designed with an interface that provides users with an easy and accesible way to utelize the model and get predictions in real-time.
Users simply need to input values for various features, such as texture mean, area mean, concavity mean, etc., through a form on the website.
Using Docker tp deploy the app ensures that the app runs seamlessly on any machine, regardless of the operating system or hardware, which aids in reproducibility and scalability.
